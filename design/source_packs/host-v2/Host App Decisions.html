Catch · Host app · Decisions & consolidation

One host app,
shaped by the lifecycle.

The host build was rebuilt twice: once faithfully on-system (the old club-shaped IA, fully route-covered), and once re-cut from first principles (the "Catch for Organizers" redesign). This consolidates them into a single direction — the redesign — and accounts for every surface so nothing is lost. It renders live from host-app-manifest.json.

Every old surface has a new home.

Each routed screen and modal surface from the old full coverage, mapped into the new IA with a status. at-risk means the redesign prototype stubbed it as a toast — the functionality must be re-homed, not dropped (see Regressions).

04 · Regressions to catch

What the redesign stubbed.

The redesign prototype is a beautiful shell, but several high-frequency flows are toasts. These are the functional regressions vs. the on-system coverage — each with the existing template that resolves it.

05 · Composed purely of catalogue widgets

Reuse what exists; promote what's missing.

The new app must be built only from catalogue components. The redesign prototype hand-rolls its widgets inline; this is the map from each element to an existing catalogue component (reuse) or a new one to promote (build).
