Catch · Host app · Information-architecture review

The host app,
re-grouped from first principles.

The host build was cleaved out of the main Catch app, and it shows: the navigation is organized around the Club, but a host's time and stakes are organized around the Event. This is a read of what exists, how often each thing is actually done, how the segment solves it, and how I'd re-cut it.

The one idea

A host lives in two objects on two cadences. The Club is a brand you set up once and tend rarely. The Event is where ~80% of host time and 100% of the high-stakes moments live — and it has a lifecycle: create → fill → run → report. Today's nav privileges the Club (two of four tabs are club-shaped) and scatters the Event lifecycle down deep nests. Flip it: make the event lifecycle the spine, and let the Club be the calm brand surface behind it.

The Club

Set-and-forget · edited a few times a year

Identity, cover, host team, payouts, the defaults that prefill events, and long-run trends. A brand object. Rarely touched once it's running.

The Event

The whole job · every week, and live on the night

Created on a cadence, filled with bookings/waitlist, run live on-site, then reported on. Every minute of real host effort is here.

01How it's organized today

Four app-bar tabs from main_host.dart. Two are club-shaped and overlap; the highest-stakes surface (running a live event) is four taps deep; analytics and your public profile are filed as sub-tabs and settings.

Events

An "operations home" that lists your clubs, each with an add-event row and upcoming-event manage rows.
Really a club→event launcher — overlaps Clubs.

Clubs

One club at a time with a switcher: Edit / Insights / Preview sub-tabs.
Analytics (Insights) is buried a tab inside a tab.

Inbox

Attendee queries & support — the consumer chat list, host-filtered.
Generic threads; not scoped to an event or a blast.

Account

Your host profile (Edit/Preview) + sign-out.
Your public face filed as a "setting."

The four symptoms of the cleave: (1) Events and Clubs overlap — both are club-organized, so related work is split across two tabs with a muddy "which one do I open?" line. (2) Running a live event — the single highest-stakes act — lives at Clubs › club › event › Manage › Live, behind the longest path in the app. (3) Two homes for "how did it go": club-level Insights in one tab, event-level Report inside Manage. (4) Profile-as-setting: the host's shareable identity sits next to sign-out instead of with the brand.

03How the segment organizes it

Two camps. Event-atomic tools (Luma, Partiful, Eventbrite) make the event the unit and pack all work into one Manage-Event hub. Community-first tools (Meetup) lead with the persistent group. Catch is genuinely a hybrid — a recurring club and an event marketplace — so it should borrow the event-hub depth from the first camp and the durable-home idea from the second.

Luma

EVENT IS THE UNIT · MANAGE = REGISTRATION / GUESTS / BLASTS / INSIGHTS / MORE · CALENDARS GROUP EVENTS
Creating an event drops you straight into one rich Manage hub; co-hosts, guests, blasts and stats all live inside the event, not scattered across tabs. The persistent grouping (Calendars) is the light Club analog.

Partiful

EVENT-FIRST & SOCIAL · ORGANIZER PROFILE GROUPS EVENTS · TEXT BLASTS · CO-ADMINS PER EVENT
The host's profile is a shareable brand with one link, and guest messaging (blasts) is first-class — both things Catch currently files away. Settings hang off a per-event gear, not a global tab.

Meetup

GROUP IS HOME · EVENTS + MEMBERS + MESSAGING HANG OFF THE GROUP
Validates a strong, durable Club object as a home — but Meetup is community-led, where Catch's energy is per-event. Take the durable home; don't let it eat the event lifecycle.

Eventbrite Organizer / Airbnb Host

EVENTBRITE: EVENTS LIST → MANAGE (DASHBOARD / ORDERS / ATTENDEES / REPORTS) · AIRBNB: TODAY / CALENDAR / LISTINGS / MESSAGES / INSIGHTS
Both lead operations with a "Today" surface and treat Messages and Insights as top-level peers. The time-sensitive now-moment gets its own home instead of being computed from a list.

04A first-principles re-cut

Re-cut the four tabs along lifecycle and cadence, not around the Club twice. Same surface count — but every five-star action moves up, and the Club becomes one calm home instead of two overlapping ones.

Today — club-shaped × 2
  • EventsClub → event launcher (overlaps Clubs)
  • ClubsEdit · Insights (analytics) · Preview
  • InboxGeneric attendee threads
  • AccountHost profile + sign-out
Proposed — lifecycle-shaped
  • Tonight / HomeTime-aware ops: the next or live event up top (one tap to run it) + "what needs you" — approvals, waitlist movement, unanswered guests, drafts
  • EventsUpcoming · Live · Past. Home of create (+ repeat last) and per-event Manage (Setup / Guests / Live / Report, roster tucked). Event analytics live in Report.
  • InboxEvent-scoped threads + a blast to all booked; requests/approvals can surface here too
  • ClubThe brand: profile (how guests see you), host team, payouts, defaults, and cross-event trends. Settings/sign-out demoted to a corner.

What moves: the live console gets a top-level fast path via Tonight; create-event leaves the club nest and lives where the events do, with a repeat-last shortcut; the host profile rejoins the brand (Club) instead of sitting in Settings; analytics consolidates — event-level in each event's Report, club-level trends in Club — instead of two disconnected homes; and the old Events/Clubs overlap collapses into one Events lifecycle + one Club brand surface.

05Highest-leverage moves

If we don't do a full re-cut, these are the changes ranked by leverage — each stands alone.

Give the live event a top-level homedo first

A "Tonight" surface (or a persistent live-event banner) that jumps straight into run-of-show. The most important moment shouldn't be the deepest screen.

Collapse Events + Clubs into Events-lifecycle + Club-branddo first

Stop running two club-shaped tabs. One Events tab owns upcoming/live/past + create; one Club tab owns identity, team, payouts, defaults, trends.

Add "repeat last event"do first

For a weekly club this is the single most-repeated creative act. One tap to clone the last run with a new date — the segment's table-stakes move (Luma/RSVPify) that Catch lacks.

Make messaging event-scoped + add a blastnext

Group threads by event and add "message all booked." Mirrors Luma/Partiful blasts; turns a generic inbox into an event-ops tool.

Move the host profile out of Settings into the brandnext

It's marketing, not a preference. Put it with the Club identity it represents; demote sign-out to a corner.

Unify analyticsnext

Event metrics in each event's Report; club trends in Club. One mental model for "how did it go," not Insights-tab vs Manage-Report.
